# BACK_STYLE 值分析经验

## 什么是 BACK_STYLE

`BACK_STYLE=num` 中的数字指向 `res/main.png`（皮肤素材图）的某个区域，决定键的背景颜色、圆角、阴影等视觉效果。

**核心原则**：不同皮肤中同一个数值的视觉效果**不一定相同**，必须针对每个皮肤单独分析。

## 分析方法

### 方法 1：查 26 键布局的对应键

```
通用规律：同皮肤内相同按键（如空格键）通常用相同 BACK_STYLE
```

如果 9 键布局的空格键样式不对，查同一皮肤的 `py_26.ini` 或 `en_9.ini` 中空格键（CENTER=F38）的 BACK_STYLE 值。26 键布局的空格通常是设计好的标准样式。

### 方法 2：查特定键的样式

常用参考键：

| 参考键 | CENTER | 特征 |
|--------|--------|------|
| 空格键 | F38 | 期望的样式 |
| 回车键 | F39 | 通常有圆角，颜色因皮肤而异 |
| 字母键 | A/B/C... | 白色小圆角 |
| 功能键 | F6/F16/F1 | 灰色/功能区样式 |

### 方法 3：批量分析 BDS

用 `scripts/analyze_bds.py` 分析 BDS 中所有 BACK_STYLE 值及其对应键。
